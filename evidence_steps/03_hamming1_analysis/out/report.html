03 汉明距离（Hamming-1）分析工具

关注点：
1) padding 到 hint 的最小汉明距离分布（越大越安全）；
2) 距离=1 的 padding/hint 对（表示 1-bit 变化就可能跨越 isHint 判定边界）；
3) padding 单比特翻转后变成 hint 的可能性（每个 padding byte 有多少个 bit 翻转能变成 hint）。

备注：在正常 TCP + AEAD 语境下，bit flip 不应无声发生；该分析主要用于说明鲁棒性边界与异常/攻击触发面。

ascii

解读要点：
1) Min dist(padding -> hint) 越大，padding 越不可能在“单/少数位变化”后被误判为 hint。
2) distance-1 pairs 表示存在 padding 与 hint 仅差 1 bit：在比特翻转/主动篡改下更容易触发误判。
3) 解码器遇到 map miss 会返回错误并触发可疑流量路径（详见 Go 实现中的 ErrInvalidSudokuMapMiss）。
