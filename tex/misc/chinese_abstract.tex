\begin{chineseabstract}
脑机接口（BCI）物联网链路同时承载高敏感神经信号与控制指令，面临重放、篡改、中间人攻击及流量侧写等复合风险。传统 TLS/DTLS 在密码学层面成熟，但在受限设备与复杂公网环境中，仍存在资源开销、部署可达性和外观可识别性等工程矛盾。针对上述问题，本文围绕仓库实现的 IoT\_BCI-Sudoku 协议，研究一种“认证握手 + AEAD 记录层 + Sudoku 外观层”组合机制，并与 Pure-AEAD、DTLS、MQTT/TLS、CoAP/UDP 在统一评估准则下对比评估。

方法上，本文在应用层与 TCP 之间实现协议增强层：握手阶段采用 Ed25519 证书签名、X25519 密钥协商、摘要绑定与确认码；会话阶段采用 AEAD 记录封装；外观层采用可逆字节映射、随机填充、ASCII 模式与自定义模式模板。为保证可证性，构建了本地基线实验、侧写与主动攻击验证实验、跨国受限网络对照实验三类实验，并给出图表到证据文件的字段级映射。

结果表明：在跨国受限网络中，Sudoku-pure 与 Sudoku-packed 的 RTT 分别为 108.676ms 和 105.536ms，与 DTLS（109.040ms）同量级且低于 MQTT/TLS（150.072ms）；数据面内存指标分别为 2712B 和 2936B，显著低于 DTLS（185472B）与 MQTT/TLS（97680B）。侧写实验中，协议识别准确率为 96.67\%；BCI 类别恢复率在无填充时为 1.0000，引入填充后降至 0.1667；TLS MITM 对照中恢复率为 1.0，而 Sudoku 重放/篡改场景均进入拒绝路径。上述结果说明 Sudoku 可在“安全、效率、抗侧写”之间形成可量化权衡：pure 更强调混淆能力，packed 更强调带宽效率。

本文贡献在于：给出 Sudoku 协议的数学与工程实现闭环，建立可复核的多环境证据链，并明确该方案相对主流基线的优势、代价与适用边界。研究结论可为 BCI-IoT 安全通信协议的工程选型与后续标准化提供依据。
