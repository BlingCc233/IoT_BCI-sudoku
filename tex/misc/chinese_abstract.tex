	
\begin{chineseabstract}
脑机接口（BCI）设备通常由信号采集端、移动端/边缘网关与服务端算法组成，数据与控制指令具有高度敏感、连续上传与低时延等特征。一旦通信链路遭受窃听、重放或中间人攻击，可能导致隐私泄露或误控制风险。现有 TLS/DTLS 等传输层安全协议在密码学层面成熟，但在资源受限设备与复杂网络环境下仍可能面临实现开销、握手成本与流量外观特征暴露等工程问题。

本文围绕本仓库实现的 IoT\_BCI-Sudoku 协议，提出一种“加密 + 外观层变换”的组合式增强思路：在不改动内核 TCP 的前提下，将协议实现置于应用层与 TCP 之间，对载荷执行身份认证握手（Ed25519 + X25519）、会话加密（AEAD record）、以及 Sudoku 外观层（映射编码、随机填充、布局轮转与可选打包）。协议同时引入抗重放机制与异常回落策略，用于降低主动探测与异常输入带来的风险。

为验证工程可用性与可复现性，本文实现了对比基线（纯 AEAD、DTLS、MQTT、CoAP/UDP）以及自动化基准评测、抓包分析与攻击模拟工具，给出在内存占用、时延、带宽开销与流量外观指标（长度/时序分布、熵值等）上的对比结果，并讨论该方案在“安全—效率—抗侧写”之间的权衡与适用边界。

\end{chineseabstract}
