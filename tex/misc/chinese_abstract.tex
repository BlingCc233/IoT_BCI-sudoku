\begin{chineseabstract}
脑机接口（BCI）物联网链路同时承载高敏感神经信号与控制指令，面临重放、篡改、中间人攻击与流量侧写等复合风险。现有 TLS/DTLS、MQTT/TLS 与 CoAP/DTLS 路线在密码学机制上较成熟，但在受限设备与复杂公网环境下仍存在资源开销、部署可达性和外观可识别性之间的工程矛盾。针对上述问题，本文围绕仓库实现的 IoT\_BCI-Sudoku 协议，研究并验证一种“认证握手 + AEAD 记录层 + Sudoku 外观层”的组合机制。

方法上，本文在应用层与 TCP 之间实现协议增强层：握手阶段采用证书签名、临时密钥协商、摘要绑定与确认码；会话阶段采用 AEAD 记录封装；外观层采用可逆字节映射、随机填充、ASCII 模式与自定义模板。为保证可证性，构建本地基线、侧写与主动攻击、跨区域受限网络对照三类实验，并建立图表到证据文件的字段级追溯关系。

实验结果显示，所提方案在受限公网条件下可保持与主流安全协议同量级的时延表现，并在数据面内存占用上具有优势；在安全验证中可稳定拦截重放与篡改输入；在侧写对抗中，填充与外观调控机制能够降低基于长度序列的类别恢复能力。进一步比较表明，Sudoku-pure 与 Sudoku-packed 分别对应“混淆优先”与“带宽优先”的不同工程取舍。

本文贡献在于：给出 Sudoku 协议从数学建模、机制设计到工程实现与证据验证的闭环；提出可复核的统一评测口径与跨环境对照流程；明确该方案相对主流基线的优势、代价与适用边界。研究结论可为 BCI-IoT 安全通信协议的工程选型与后续标准化提供参考。
