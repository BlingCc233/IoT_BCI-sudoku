\begin{englishabstract}
Brain-computer interface (BCI) IoT links carry highly sensitive neural data and control commands, and are exposed to replay, tampering, man-in-the-middle (MITM), and traffic-fingerprinting attacks. To address the conflict among constrained devices, public-network reachability, and traffic observability, this thesis implements and validates a composite secure protocol that combines an authenticated handshake, AEAD records, and a reversible Sudoku-based appearance layer.

The protocol is implemented as an enhancement layer between the application and TCP. The handshake integrates certificate-based authentication, ephemeral key agreement, transcript binding, and confirmation MAC; the session layer adopts AEAD records; the outer appearance layer provides reversible byte mapping and randomized padding, with a configurable trade-off between the full Sudoku appearance mode and a 6-bit segmented mode. The evaluation pipeline includes local baseline benchmarking, fingerprinting and active-attack validation, and cross-region restricted-network experiments under unified metrics and repeated-run statistics.

The evaluation shows that the proposed scheme maintains latency at the same practical level as mainstream secure protocols under restricted-network conditions, while exhibiting lower data-plane memory usage. In security validation, replay and tampering inputs are consistently rejected. In side-channel evaluation, MQTT/TLS, CoAP/UDP, Pure-AEAD, and the unpadded Sudoku mode all show 100\% class recoverability, whereas randomized padding in the full Sudoku appearance mode reduces the recovery rate to 16.67\%. The cost is higher byte-stream overhead and lower bandwidth utilization.

The core contribution is a validated composite protocol built from authenticated handshake, AEAD records, and a reversible Sudoku-based appearance layer. The results show that the design reduces passive fingerprinting recoverability while keeping latency in the same order as mainstream protocols, with explicit bandwidth cost and applicability boundaries. The results provide a practical reference for protocol selection and future standardization in secure BCI-IoT communication.

\englishkeyword{BCI, IoT secure communication, Sudoku coding, random padding, traffic fingerprinting}
\end{englishabstract}
