\section{指标定义与统计规则}
\subsection{性能指标}
\begin{itemize}
  \item RTT：\path{avg_rtt_ms} 与 \path{p95_rtt_ms}；
  \item 开销：\path{overhead_ratio = wire_bytes_total/payload_bytes_total}；
  \item 吞吐：\path{payload_bps} 与 \path{wire_bps}。
\end{itemize}

\subsection{内存指标}
\begin{itemize}
  \item 绝对峰值：\path{peak_heap_inuse_bytes}；
  \item 数据面守卫：\path{phase_delta_heap_alloc_bytes}。
\end{itemize}

第二指标定义用于跨国守卫判定，并与跨国评测脚本的统计字段保持一致，以保证结果可复核性。

\subsection{统计规则}
本文统一采用重复运行与中位数统计准则。跨国 step07 默认 \path{runs=3}，本地 step05 默认 \path{repeats=3}。若出现单次异常值，不以均值直接下结论。

\section{侧写算法复现要点}
step06 的协议侧写采用最近质心留一验证。核心过程为：
\begin{itemize}
  \item 从长度序列、到达间隔、熵值、可打印比例提取特征向量；
  \item 逐样本执行 leave-one-out；
  \item 计算混淆矩阵与总体准确率。
\end{itemize}

相关脚本：\path{evidence_steps/06_fingerprint_sidechannel/run.py}，核心函数为 \path{feature_from_metrics} 与 \path{nearest_centroid_loo}。

\section{跨国实验部署约束}
跨国实验采用本机客户端与加拿大区域公网云主机。为保护运维资产安全，正文与附录均不公开服务端公网 IP。受公网策略影响，UDP 直连不可用：
\begin{itemize}
  \item Sudoku / Pure-AEAD / MQTT-TLS：TCP 直连；
  \item DTLS / CoAP：UoT（UDP-over-TCP）中继。
\end{itemize}

该约束已写入 \path{evidence_steps/07_cross_region_benchmark/out/summary.json:notes.udp_transport}。
