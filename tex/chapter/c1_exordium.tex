\chapter{绪论}

\section{研究背景与意义}
脑机接口（Brain-Computer Interface, BCI）在医疗康复、人机交互与智能控制等场景中逐步落地。典型的 BCI 系统包含信号采集端、移动端/边缘网关与服务端算法，采集端需要连续上传高频数据，网关与服务端还会下发控制指令或参数更新。该类通信具有三类共性约束：\textbf{数据敏感}（隐私与生物特征）、\textbf{实时性强}（低时延/低抖动）、\textbf{资源受限}（端侧内存与功耗）。

在开放网络环境下，攻击者可对链路进行窃听、重放与篡改（MITM），也可通过包长、时序等外观特征对协议与业务进行侧写推断。TLS/DTLS 等标准协议在密码学安全性方面成熟，但在受限设备与复杂网络中仍面临握手成本、状态开销与流量指纹暴露等工程挑战。因此，面向 BCI-IoT 场景，需要研究一种兼顾\textbf{认证与抗重放}、\textbf{轻量化实现}、并能降低\textbf{流量外观可辨识性}的通信增强方案。

\section{问题定义与需求约束}
本文关注 BCI-IoT 端到端安全通信的工程落地，需求可归纳为：
\begin{itemize}
  \item \textbf{安全性}：抵御窃听、重放与中间人攻击；异常输入下不泄漏敏感响应。
  \item \textbf{资源与性能}：端侧内存占用可控，单位收发处理延迟低，连接状态尽量小。
  \item \textbf{抗侧写}：在不削弱机密性与完整性的前提下，降低基于长度/时序/字节分布的可观测特征。
  \item \textbf{可复现}：评测流程、抓包分析与安全验证能够自动化复现并输出量化指标。
\end{itemize}

\section{本文工作与贡献}
围绕上述需求，本文基于仓库实现了 IoT\_BCI-Sudoku 协议与评测工具链，主要工作包括：
\begin{itemize}
  \item \textbf{协议设计}：提出“加密 + 外观层变换”的分层结构，包含身份认证握手（Ed25519 + X25519）、AEAD 会话记录层，以及 Sudoku 外观层（编码映射、随机填充、布局轮转与可选打包）。
  \item \textbf{威胁模型与应对}：建模重放、MITM 与资源耗尽等威胁，给出抗重放窗口、握手绑定与异常回落策略。
  \item \textbf{工程实现}：提供可复用 Go 协议库与示例节点程序，支持 stream/mux/UoT 等应用形态。
  \item \textbf{对比评测}：实现对比基线（纯 AEAD、DTLS、MQTT、CoAP/UDP），统一输出内存/时延/带宽开销与外观指标（熵值、长度/时序分布等）。
  \item \textbf{攻击模拟与抓包侧写}：提供重放、篡改与探测洪泛的模拟工具，并基于抓包结果进行协议指纹识别与对比分析。
\end{itemize}

\section{论文结构}
本文组织结构如下：第 2 章介绍相关研究与工程背景；第 3 章给出协议分层、握手与外观层的设计与实现细节；第 4 章描述实验方法、指标口径与对比结果，并分析外观层在抗侧写方面的表现；第 5 章总结全文并给出后续工作展望。
