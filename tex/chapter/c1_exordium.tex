\chapter{绪论}

\section{研究背景与问题提出}
脑机接口（Brain-Computer Interface, BCI）系统作为连接大脑与外部设备的直接通路，正在从早期的实验室探索逐步走向广泛的实际应用。根据信号采集方式的不同，BCI 系统主要分为侵入式、半侵入式和非侵入式三类。其中，非侵入式 BCI 由于其安全、便捷、低成本的特点，在医疗康复、人机协同、智能控制以及娱乐交互等领域展现出巨大的应用潜力\cite{meng2024adversarial,lei2017bcireview}。随着物联网（Internet of Things, IoT）技术的飞速发展，将 BCI 设备接入互联网络，形成 BCI-IoT 融合系统，已成为当前学术界和工业界的研究热点。在这种融合架构下，位于边缘侧的脑电采集设备通过无线网络实时上传生理信号至云端或边缘服务器进行处理，同时接收来自服务端的反馈控制指令，从而实现远程监测与闭环控制。

然而，BCI-IoT 系统的广泛部署面临着严峻的安全与隐私挑战。由于脑电信号包含极其敏感的用户生物特征与意图信息，一旦泄露或被篡改，可能导致严重的隐私侵犯甚至人身安全事故。在典型 BCI-IoT 架构中，采集端、边缘网关与云端算法服务形成闭环：上行链路持续传输生理信号，下行链路承担参数更新与控制指令。该场景对通信协议提出了三类并行约束：数据敏感性高、交互时延容忍度低、端侧资源受限\cite{hua2018iotsecurity,zhang2017iotsurvey}。物联网设备通常计算能力有限、内存空间狭小且电池容量受限，难以支持复杂的传统安全协议；同时，无线传输环境的开放性使得通信链路极易遭受窃听、干扰和中间人攻击。

单纯依赖“内容加密”难以覆盖 BCI-IoT 的全部风险。攻击者在密文条件下仍可利用包长、方向、到达间隔和字节统计特征实施流量侧写\cite{velan2015survey,pan2016encryptedtrafficcn,hou2024encmalware}；在主动攻击维度，重放、篡改与中间人攻击会直接影响控制安全与系统可用性。基于此，本文研究问题可归纳为：如何在不改动内核 TCP 的前提下，同时实现密码学安全、资源可控与外观抗侧写。

\section{系统模型与威胁模型}
\subsection{系统模型}
本文研究对象为位于应用层与 TCP 之间的传输增强层。受保护资产包括业务明文（BCI 类别、控制参数）、会话密钥材料、认证状态以及可被学习的流量外观特征。系统需支持连续上传与指令回传并发存在的业务模式。

\subsection{威胁模型}
本文采用“被动观测 + 主动干扰”的组合威胁模型。被动攻击者可长期抓取流量并训练分类模型；主动攻击者可重放历史报文、篡改握手字段或注入探测数据。本文不假设攻击者能够直接破解标准密码算法，而重点关注协议工程层面的可识别性与异常路径安全。

\subsection{研究假设与边界}
为保证论证可检验，本文明确以下假设：
\begin{itemize}
  \item \textbf{A1（密码原语安全）}：不讨论标准密码算法被直接攻破情形，重点分析协议工程与实现路径风险；
  \item \textbf{A2（端侧资源受限）}：设备侧内存与 CPU 预算有限，协议状态必须有上界；
  \item \textbf{A3（公网受限可达性）}：真实部署中可能存在 UDP 受限、代理终止与策略审查\cite{rfc8085,rfc4787,li2011nat}；
  \item \textbf{A4（可观测外观）}：攻击者可长期获得长度、时序、方向与字节分布等统计特征。
\end{itemize}

据此，本文的论证目标定义为“可部署约束下的风险收敛与可复核改进”，即在同等部署条件下减少可利用攻击面，并量化性能与开销代价\cite{rfc8446,rfc9147}.

\section{研究目标与研究问题}
围绕“安全性、可部署性、抗侧写”三维目标，本文提出四个研究问题：
\begin{itemize}
  \item \textbf{RQ1（轻量可用性）}：如何在资源受限设备上控制时延与内存占用；
  \item \textbf{RQ2（安全机制落地）}：如何将抗重放与抗篡改机制落地为可执行握手流程；
  \item \textbf{RQ3（可逆外观编码）}：如何保持 Sudoku 外观编码可逆且可调；
  \item \textbf{RQ4（可证性评估）}：如何在统一评估准则下得到稳定、可复核的对比结论。
\end{itemize}

针对 RQ4，本文采用如下判定规则。记 Sudoku 方案为 pure 模式：
\[
\begin{aligned}
G_{\text{pure}} &= \mathbf{1}\!\left(\mathrm{RTT}_{\text{pure}} < \min(\mathrm{RTT}_{\mathrm{DTLS}}, \mathrm{RTT}_{\mathrm{MQTT}})\right) \\
&\times \mathbf{1}\!\left(\mathrm{MEM}_{\text{pure}} < \min(\mathrm{MEM}_{\mathrm{DTLS}}, \mathrm{MEM}_{\mathrm{MQTT}})\right).
\end{aligned}
\]
其中 MEM 取峰值内存指标，overhead 允许高于基线。

\section{研究假设与可验证命题}
围绕上述研究问题，本文进一步给出四个可验证命题：
\begin{itemize}
  \item \textbf{H1}：在跨国受限网络下，Sudoku 的 RTT 与主流协议处于同量级；
  \item \textbf{H2}：在相同工作负载下，Sudoku 的数据面内存增量低于 DTLS 与 MQTT；
  \item \textbf{H3}：引入填充与外观编码后，类别侧写恢复率显著下降；
  \item \textbf{H4}：Sudoku 在主动重放与篡改场景下可稳定进入拒绝路径。
\end{itemize}

第 4 章将按“证据文件 $\rightarrow$ 指标定义 $\rightarrow$ 统计结论”三步验证 H1--H4。若任一命题不成立，将在结论章节给出反例条件与失败边界，不进行选择性报告。

\section{本文主要工作与创新点}
本文完成的主要工作如下：
\begin{itemize}
  \item 设计并实现面向 BCI-IoT 的 Sudoku-pure 协议原型，形成“握手认证 + AEAD 记录 + 外观层”分层结构；
  \item 给出可逆映射机制，实现字节到 Sudoku 提示序列的稳定映射与冲突检测；
  \item 构建异常路径处置机制，使重放与篡改输入进入可疑流量处理路径；
  \item 建立统一实验工具链，完成与 DTLS、MQTT、Pure-AEAD、CoAP 的对照评测；
  \item 构建侧写与 MITM 对照实验，验证外观层在类别保护与抗篡改上的作用。
\end{itemize}

本文创新点主要体现在三个方面：一是将密码学安全与外观层对抗进行组合设计；二是给出 Sudoku 外观编码的可逆数学表达与复杂度分析；三是建立覆盖性能、外观与攻击验证的一体化实证框架。与仅做概念论证的方案不同，本文全部结论都落在可运行实现与可复现实验脚本上，形成“理论--实现--证据”的闭环。
