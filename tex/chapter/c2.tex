\chapter{相关研究与理论基础}

\section{BCI-IoT 通信需求与评价维度}
BCI-IoT 链路同时承载连续脑电数据与控制指令，具有“高敏感数据、低时延交互、长期在线运行”的复合特征\cite{lei2017bcireview,hua2018iotsecurity}。因此，协议评价不能只看“是否加密”，还需同时回答三个问题：其一，安全机制是否完整；其二，资源开销是否可控；其三，外观特征是否会被侧写模型稳定学习。

本文据此采用四个评价维度：安全建立过程、会话保护能力、资源占用与外观可识别性。后续对比均在统一消息规模、统一统计定义与重复实验中位数规则下开展。该评估准则与 IoT 场景中关于 TLS/DTLS 性能研究所强调的“同负载、同实现、同统计”原则一致\cite{restuccia2020lowpower,gallenmueller2019dtls}。

\section{主流协议通信机理与安全实现}
\subsection{Pure-AEAD 基线}
本文所称 Pure-AEAD（纯记录层 AEAD）是“仅记录层加密”的最简方案：双方预共享密钥（PSK）派生会话密钥与盐值，再按记录格式逐条执行 AEAD 加密/解密\cite{rfc5116}。在该路径中，机密性与完整性来自 AEAD 标签验证；但身份认证、新鲜性与重放窗口等能力不由记录层接口自动提供，而需上层协议自行补足。其工程优点是实现轻、状态少、吞吐高；局限是安全职责集中在上层状态机，设计不当时更易出现重放或身份混淆边界。

\subsection{MQTT/TLS 基线}
MQTT 是发布/订阅应用协议，本文实现过程为：设备端与服务端先建立 TCP 连接，再完成 TLS 1.3 握手，随后执行 MQTT 的 \texttt{CONNECT/CONNACK}、\texttt{SUBSCRIBE} 与 \texttt{PUBLISH} 交互\cite{mqtt311,rfc8446}。TLS 负责证书认证、密钥协商与记录加密，MQTT 负责主题路由与消息分发语义。该路线工程成熟、生态丰富，但系统由“传输安全层 + 消息中间件层”叠加构成，实现链路长，且加密后外观特征仍可被侧写模型利用。

\subsection{CoAP/DTLS 基线}
CoAP 是面向受限设备的报文协议，运行在 UDP 之上，使用 \texttt{CON/ACK}、消息 ID 与令牌实现应用层请求匹配与重传控制\cite{rfc7252}。为获得加密与认证，常与 DTLS 叠加：先完成 DTLS 握手，再承载 CoAP 业务报文\cite{rfc9147,rfc7925}。该方案对短报文友好，但公网策略、NAT 与防火墙常使 UDP 可达性成为首要约束；在受限网络中，协议可达性问题往往先于密码学机制问题暴露。

\subsection{Sudoku-pure 方案}
Sudoku-pure 采用“认证握手 + 会话记录 + 外观层编码”三层结构。握手阶段使用证书签名、临时密钥协商、摘要绑定与确认码，保障身份、密钥一致性和握手新鲜性；会话阶段使用 AEAD 记录层；最外层将字节编码为数独提示字节序列，并可注入填充字节以改变外观统计特征。该方案定位于 BCI-IoT 场景下“主动攻击面压缩 + 侧写可学习性抑制”的联合优化。

\subsection{协议状态机与安全职责分层}
从实现责任看，四类路线在“安全能力由谁提供”上存在明显差异：
\begin{itemize}
  \item Pure-AEAD：记录层负责机密性/完整性；身份认证与防重放由上层自管；
  \item MQTT/TLS：TLS 负责握手安全与密钥；MQTT 负责会话语义与路由；
  \item CoAP/DTLS：DTLS 负责握手与会话保护；CoAP 负责报文匹配与重传；
  \item Sudoku-pure：握手状态机提供身份与新鲜性，记录层提供机密性，外观层提供抗侧写调控。
\end{itemize}

这一区分对于论文判据有直接影响：当“重放可拒绝、外观可调控、资源有上界”是联合目标时，单层机制往往不足，需在状态机、记录层与外观层之间显式分工。

\section{IoT 安全协议性能实证与研究空缺}
TLS/DTLS 在受限设备上的性能代价已有较多实证。Restuccia 等在低功耗 IoT 设置中比较了 TLS 1.3 与 DTLS，指出握手与记录层引入的状态管理会显著影响端侧资源\cite{restuccia2020lowpower}；Gallenm{\"u}ller 等通过分解式测量进一步表明，安全机制带来的额外往返与实现开销对短报文业务尤为敏感\cite{gallenmueller2019dtls}。这些研究为本文第 4 章的性能判据提供了经验依据。

但现有工作仍存在三点空缺。其一，多数研究聚焦“加密成本”，较少将“流量外观可学习性”与性能指标并行讨论；其二，公开评测通常停留在实验室网络，缺少受限公网条件下的复核；其三，关于 BCI 场景的研究更多关注算法模型与数据安全\cite{kapitonova2022framework,meng2024adversarial}，较少提供“协议机制--实现--攻击验证”闭环证据。本文将这三类空缺作为后续章节的重点补全目标。

\section{流量侧写与对抗研究}
现有研究已证明，即使明文不可见，长度、方向、到达间隔、字节熵等统计特征仍可支持高精度分类\cite{velan2015survey,hayes2016kfingerprinting,sirinam2018deepfingerprinting,wang2017walkietalkie}。本文采用与 Aparecium 类工具一致的流程：先抽取长度与序列特征，再执行协议识别与类别恢复评估（\url{https://github.com/ban6cat6/aparecium}）。

此外，本文引入 TLS 中间人代理解密对照，用于刻画特定运维/攻防条件下终止代理点的可见性风险，即“链路加密已启用但业务仍可恢复”的现实边界\cite{wilkens2021passivetls}。

\section{Sudoku-pure 数学基础与术语}
\subsection{术语定义}
为减少歧义，本文统一采用以下中文术语：
\begin{itemize}
  \item \textbf{提示字节}：编码后可被解码器识别为“数独提示位”的字节；
  \item \textbf{填充字节}：仅用于外观扰动、解码时被忽略的字节；
  \item \textbf{模式模板}：定义字节位上“冗余位/位置位/取值位”分布的 8 位模式；
  \item \textbf{解码键}：4 个提示字节按序拼接得到的 32 位键值。
\end{itemize}

在实现与论文叙述中，本文将“提示字节/填充字节/模式模板”界定为功能术语。安全性论证建立在标准握手、密钥派生、AEAD 与状态机正确性之上，外观编码用于外观调控而非密码学能力替代。

4$\times$4 Sudoku 解空间定义为
\[
\mathcal{S}=\{g\mid g \text{满足行、列与 }2\times2\text{ 子宫格约束}\},
\]
其规模为 $|\mathcal{S}|=288$，可选取其中 256 个解与字节集合建立一一对应\cite{oeisA107739}。

\subsection{组合空间与排列基数}
从组合数学角度看，若按会话种子从 288 个可行网格中为 256 个字节选择互异映射，则可行映射上界为排列数
\[
P(288,256)=\frac{288!}{32!}\approx 2^{1825}.
\]
该值用于刻画外观映射重排的数量级上界，可反映离散映射空间规模。进一步地，每个字节的提示位位置组合满足
\[
|\mathcal{C}|=\binom{16}{4}=1820,
\]
与填充注入共同作用时，会显著扩大可观测外观形态数量。该分析与 Sudoku 相关组合研究“搜索空间极大、约束主导求解复杂度”的结论一致\cite{lynce2006satsudoku,mcguire2014no16clue,herzberg2007sudoku}.

\subsection{可逆映射与复杂度}
对字节 $b\in\mathcal{B}$，先映射到网格 $g_b=f_k(b)$，再从网格中抽取 4 个提示位编码为 4 个提示字节。解码时按 4 字节拼接形成解码键，通过映射表 $D$ 还原原字节。构表阶段若发现不同原字节产生同一解码键，则直接拒绝构表，因此可保证单值可逆。

在 ASCII 布局下，提示字节可写为
\[
h = 0x40 \,|\, (v \ll 4)\,|\, p,
\]
其中 $p\in[0,15]$ 为位置编码，$v\in[0,3]$ 为取值编码。编码与解码对长度为 $n$ 的消息均为线性复杂度 $O(n)$。

无填充时，若第 $i$ 个字节候选提示组数量为 $K_i$，可观测形态规模为
\[
N_{\text{无填充}}=\prod_{i=1}^{n}K_i\approx 8^n.
\]
若每字节存在 5 个填充注入检查点，报文尾部 1 个检查点，填充池大小为 $M$，则
\[
N_{\text{线上}} = N_{\text{无填充}}(M+1)^{5n+1}.
\]
该表达式解释了“外观随机性增强”与“带宽开销上升”之间的必然权衡。

\subsection{Sudoku 握手安全性的比较说明}
与 Pure-AEAD 相比，Sudoku 在握手阶段显式引入证书签名、双向随机数回显、摘要绑定与确认码，安全收益体现在三个方面：
\begin{itemize}
  \item \textbf{身份约束}：握手消息均与证书及签名绑定，降低身份伪造风险；
  \item \textbf{上下文绑定}：ClientHello/ServerHello 摘要进入确认码计算，降低反射与转发攻击成功率；
  \item \textbf{新鲜性约束}：随机数与时间窗口进入重放检测令牌，历史握手难以直接复用。
\end{itemize}

与 TLS/DTLS 相比，Sudoku 的改进重点在于将外观调控能力纳入同一协议状态机，并保持资源上界可控。基于该定位，本文将其归类为“工程折中路线”。

\section{统一评测指标与复现规则}
本文采用如下统一指标：
\[
R_{\text{开销}}=\frac{B_{\text{wire}}}{B_{\text{payload}}},
\quad
\overline{\mathrm{RTT}}=\frac{1}{N}\sum_{i=1}^{N}\mathrm{RTT}_i,
\quad
\mathrm{P95}=\operatorname{Quantile}_{0.95}(\{\mathrm{RTT}_i\}),
\]
\[
H=-\sum_{x=0}^{255}p_x\log_2 p_x,
\quad
\rho_{\text{可打印}}=\frac{\sum_{x=0x20}^{0x7E}c_x}{\sum_{x=0}^{255}c_x},
\quad
T_{\text{payload}}=\frac{B_{\text{payload}}}{t}.
\]
其中 $H$ 为字节熵，$\rho_{\text{可打印}}$ 为可打印字节比例。所有关键结论均要求“多次运行 + 中位数 + 稳定性检查”，避免单次抖动主导结果。

\section{本章小结}
本章给出了后续实验所需的理论与工程背景：说明了 Pure-AEAD、MQTT/TLS、CoAP/DTLS 与 Sudoku-pure 的通信流程和安全实现差异，明确了侧写风险语境，统一了“提示字节、填充字节、模式模板”等术语，并给出可逆映射与复杂度表达。这些内容构成第 3 章实现与第 4 章对比分析的基础。
