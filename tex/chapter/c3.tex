\chapter{协议设计与实现}

\section{总体架构与分层}
IoT\_BCI-Sudoku 以 TCP 为承载，不修改内核协议栈，而是在应用层与 TCP 之间叠加一组可组合的增强层。其核心分层为：
\begin{itemize}
  \item \textbf{握手认证层}：完成身份认证、密钥协商/派生与参数协商。
  \item \textbf{会话记录层（可选）}：以 AEAD 将长连接上的数据流帧化为 record，提供机密性与完整性保护。
  \item \textbf{外观层（Sudoku）}：对 record 或上层 payload 进行映射编码、随机 padding 与布局轮转，降低可观测特征。
  \item \textbf{应用层}：支持 stream（长度前缀帧）、mux（多路复用）与 UoT（datagram over stream）三类形态，便于对接不同业务。
\end{itemize}

上述分层以“可选项”形式组合，便于在不同设备资源与网络条件下进行权衡与裁剪。

\section{威胁模型与安全目标}
本文考虑的攻击者能力包括：被动窃听、主动篡改与注入、重放历史报文、以及通过外观特征进行侧写推断。对应安全目标为：
\begin{itemize}
  \item \textbf{机密性与完整性}：会话数据对窃听者不可读，对篡改可检测。
  \item \textbf{抗重放}：历史报文重放不会被当作新指令执行，且不会触发可被利用的差分响应。
  \item \textbf{抗 MITM}：握手阶段完成身份绑定与密钥派生的验证，降低中间人伪装成功概率。
  \item \textbf{抗侧写（工程目标）}：降低长度/时序/字节统计特征的稳定性，从而降低自动识别与推断的可行性。
\end{itemize}

\section{握手认证与密钥派生}
握手阶段采用 Ed25519 作为身份认证签名算法，并使用 X25519 生成会话的临时密钥材料。握手消息包含时间戳、随机数（nonce）、双方公钥/证书信息与签名，用于实现身份绑定与重放检测。会话密钥通过 HKDF 结合握手 transcript hash 派生，确保密钥与本次握手上下文绑定。

为了降低主动探测风险，握手阶段也可包裹一层预共享密钥的 AEAD（握手 record），使得未知密钥的探测者难以构造可被正确解析的握手数据。

\section{抗重放与异常回落}
抗重放策略由两部分组成：一是时间戳窗口（TimeSkew）限制，二是基于 token 的 replay cache。握手在认证通过后才提交 token，使得攻击者难以通过伪造报文污染缓存。对于异常输入与可疑流量，协议实现采用一致化错误响应与回落策略：尽量不暴露可区分的错误细节，并允许上层将连接转入“沉默/降级”路径。

\section{Sudoku 外观层：映射、padding 与轮转}
外观层的目标是在“已加密”的前提下，进一步降低外观特征的稳定性。本文选取 4$\times$4 Sudoku 作为编码载体：其非同构体数量为 288，大于单字节的 256 取值空间，能够用于构造可逆映射。实现上，将每个字节映射为 4 个 hint，并通过：
\begin{itemize}
  \item \textbf{随机 padding}：引入可配置的填充长度，使得整体写入大小不固定；
  \item \textbf{布局轮转}：对字节布局进行轮转/置换，使得相同 payload 在不同会话中呈现不同外观；
  \item \textbf{可选打包（packed）}：对下行小包进行聚合，降低写调用次数与边界开销；
\end{itemize}
从而改变长度分布、写入粒度与时序分布，降低指纹的稳定性。

\section{工程实现要点}
协议核心以 Go 实现并模块化为可复用库（\texttt{pkg/iotbci} 与 \texttt{pkg/obfs/sudoku}）。工程实现的关注点包括：端侧内存占用、热路径上的分配控制、以及异常输入下的状态机鲁棒性。为支持复现实验与对比评估，本仓库提供了基准跑分、证据集生成、抓包分析、攻击模拟与统一可视化等工具链（\texttt{cmd/iotbci-*}）。
